# hint 1.0.0 — pure extraction

Extract the copied-around `substrate.html` HTML-builder into a standalone, CI'd, installable library
(`hint`, distributed as `hint-html`), widening the element set to the full HTML vocabulary. Streaming
(1.1.0) and the other extractions are follow-up milestones, not part of this mega-PR.

Design spec: `docs/superpowers/specs/2026-07-10-hint-extraction-design.md`.

## Plan

One `###` per planned sub-PR. Each is a *problem statement* — the objective and why it's its own slice.
The "how" is decided by surveying the code when the sub-PR is reached, and written up in that sub-PR's own
description. Re-slice freely; headings are descriptive so inserting/splitting a section is a non-event.

### Package scaffolding & build

Branch: `feat/extraction-scaffold`

Make `hint` an installable, importable, typed package with the tooling *config* in place (enforcement lands
with CI later).

- [x] `pyproject.toml`: dist `hint-html`, import `hint`, Python 3.14 floor, **setuptools** (house style,

- [x] Locked deps (`uv.lock`), minimal importable `src/hint/__init__.py`, `py.typed`
- [x] Tool config: ruff (`select = ["ALL"]`, lint + format), pyright (strict), import-linter; plus a
      `Makefile` and `.python-version` matching house style
- [x] `.gitignore` (`.idea/`, `.locked-branches`, build/venv artefacts)

### Core: element tree, render, render_html

Branch: `feat/extraction-core`

Port the canonical dataclass core and the render step, factored so 1.1.0 streaming can slot in without a
rewrite.

- [x] `RawHtml`, `Element`, `ElementOrStr`, `element`/`Node`, `style`
- [x] `render` — escapes text + attribute values, handles void elements; recursive walk (str-returning),
      streaming-amenable without building the generator now (YAGNI until 1.1.0)
- [x] `render_html` — prepends doctype, rejects non-`html` root
- [x] Tests: escaping never bypassed (hypothesis), structure round-trips, void vs non-void,
      `render_html` root check, `RawHtml` pass-through, `style` verbatim

### Full HTML element vocabulary

Branch: `feat/extraction-tags`

Export the full current HTML Living Standard element set as `Node` constructors.

- [x] All current elements — 111 constructors (obsolete excluded; SVG/MathML excluded — BACKLOG);
      `<style>` stays the `style()` helper, not a plain constructor
- [x] `del_` alias for `<del>`; file-scoped `# ruff: noqa: A001` for builtin-shadowing tag names, with rationale
- [x] Official void-element set (landed with core) drives self-closing render
- [x] Tests: representative sample across categories, `del_` → `<del>`, builtin-shadowing tags, void self-close

### Optional Markdown rendering

Branch: `feat/extraction-markdown`

Optional `markdown()` that costs nothing unused and is resolved once.

- [x] `hint._markdown`: attempt `markdown_it` import once at load, bind a strategy (no per-call check, no boolean)
- [x] present → `RawHtml(rendered)`; absent → `pre([text], {})`; re-exported as `hint.markdown`
- [x] safe-by-default: parser uses `html=False` (raw HTML in the markdown *input* is escaped, not passed
      through), after a security review flagged the original `html=True` as an XSS vector for untrusted input
- [x] `import-linter` contract: `markdown_it` reachable only via `hint._markdown` (verified it breaks on a
      forbidden core import); core stays third-party-free
- [x] Tests: present branch (real dep) + absent branch (resolver under `sys.modules` monkeypatch)

### Docs: README & BACKLOG

Branch: `feat/extraction-docs`

- [x] `README.md`: rehomed "why code, not templates" rationale + API, install/usage, the `tag([children], {attrs})`

## Scope

**In:** the six slices above → a complete, CI'd, documented, typed `hint` 1.0.0 with the full HTML vocabulary
and optional Markdown.

**Out (deferred → BACKLOG.md):** streaming `render_stream` (1.1.0); content negotiation (`substrate.negotiate`);
pagination helpers; canonical query parameters; chrome/layout vocabulary; SVG/MathML; PyPI publish + mypy gate.

**Out (other repos / follow-up):**
- Editing `~/Config/claude/html-rendering-pattern.md` to point at `hint` (it lives in the `~/Config` repo — a
  separate change there, not a sub-PR here). Tracked under References.
- Migrating consumers (`yog-sothoth`, `quick-portraits`, `easel-prompt`, `notes-jubilant-guacamole`) off their
  vendored `substrate.html` onto `hint` — per-project follow-up milestones.

## Design Decisions

See the spec. Key settled points: calling convention `tag([children], {attrs})` is unchanged and deliberate;
faithful superset, not a redesign; pyright over mypy for now; full HTML vocabulary; markdown resolved once via
a bound strategy, not a boolean or per-call check.
